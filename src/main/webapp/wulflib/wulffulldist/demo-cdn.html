CDN

WULF can be included into your project dynamically in runtime using CDN solution. You find the addresses below for HTTP protocols.

HTTP
For OT31:
http://wulf-cdn.dynamic.nsn-net.net/OT31/latest/css/wulf.min.css
http://wulf-cdn.dynamic.nsn-net.net/OT31/latest/js/wulf.min.js

For OT40:
http://wulf-cdn.dynamic.nsn-net.net/OT40/latest/css/wulf.min.css
http://wulf-cdn.dynamic.nsn-net.net/OT40/latest/js/wulf.min.js

For specific version, e.g. WULF_OT31_1_2_13:
http://wulf-cdn.dynamic.nsn-net.net/WULF_OT31_1_2_13/css/wulf.min.css
http://wulf-cdn.dynamic.nsn-net.net/WULF_OT31_1_2_13/js/wulf.min.js

Benefit of this is that you don't need to download WULF / re-package your web pages to get a new version into use. You have always the latest version of WULF in use automatically. This is handy for making quick proof of concepts and small internal web tools. At the moment we are not offering any frozen WULF versions via CDN, it is always the latest development version - if certain released versions offering is needed via the CDN, let us know.

Note: this CDN approach is not directly feasible for web applications given to customers. As the WULF CDN server is located inside Nokia intranet the applications cannot access the files from outside.


Usage

You just replace in your web pages:
<link href="./wulfdist/css/wulf.css" rel="stylesheet"/>
...
<script src="./wulfdist/js/wulf.min.js"></script>
with for example: 
<link href="http://wulf-cdn.dynamic.nsn-net.net/OT31/latest/css/wulf.css" rel="stylesheet"/>
...
<script src="http://wulf-cdn.dynamic.nsn-net.net/OT31/latest/js/wulf.min.js"></script>
