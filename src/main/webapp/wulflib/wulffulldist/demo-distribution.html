Instruction for distribution folder structure change

wulffulldist/wulfdist: Distributed for jquery components, that is you can copy this folder for jquery based project.

  • CSS: Include style definition for all wulf components, it is common for all projects.
  • Fonts/Img: It is needed for the css style definition.
  • JS: Include all js files that is needed for wulf components.
    • components: seperated js file for each component
    • dependency: the 3rd part lib wulf depends on.
    • wulf.min.js: combine all seperated js files as one
    • wulf-concat-deps.min.js: wulf.min.js + depended 3rd lib(exclude jqxWidget since the size seems large, suggest import by youself if needed).

wulffulldist/wulfdist_ang15: Distributed for angular components, that is you can copy this folder for angular based project.

  • CSS/Fonts/Img: Same with the folder in wulfdist, they are common for projects.
  • JS: Include angular related js and template files.
    • components: component js file, it need to work with template together; combined component and template files(suffix with *.tlp.min.js), that is no dependency on template file.
    • template: it is the component template file. it needs to work with component js file together.
    • wulf.angular.min.js: combine all component js files for angular but not include template file, it is minimized.
    • wulf.angular.tlp.min.js: combine all component js files and corresponding template files, it is minimized.

wulffulldist/wulfdist_non_minified: All the files in this folder are not minimized, that is you can check the detail of css/js files.

  • CSS: Include css files for seperated component(wullf.*.css) and for all components(wulf.css). it is common for jquery based components and Angular based components.
  • JS: Include JQuery based js files for seperated component and for all components(wulf.js / wulf-concat-deps.js).
  • ang15: Include Angular based js files and template files for each component.
    • JS: seperated js file for the angular component, it needs to work with template together.
    • template: it is the component template file.
    • wulf.angular.js: it is a single file that include all components for angular. The template is still needed.
    • wulf.angular.tlp.js: it is a single file that include all components and template files.

Instruction for dist folder

  • CSS: Include style definition for all wulf components, it is common for all projects.
  • Fonts/Img: It is needed for the css style definition.
  • JS: include JQuery based js files and Angular based js files.
    • wulf.js/wulf(.min).js: Combine all seperated component js files as one for JQuery based.
    • wulf-concat-deps(.min).js: wulf(.min).js + depended 3rd lib(exclude jqxWidget since the size seems large, suggest import by youself if needed).
    • angular15: Angular component related files.
      • components: component js file, it need to work with template together; combined component and template files(suffix with *.tlp.min.js), that is no dependency on template file.
      • template: it is the component template file. it needs to work with component js file together.
      • wulf.angular.min.js: it is a single file that include all components for angular. The template is still needed.
      • wulf.angular.tlp.min.js: it is a single file that include all components and template files.

Instruction for how to use angular

Let's use wulfdist_ang15 as example, it is same for bower update(by using dist). There are four ways to support angular with wulf, you can decide to choose one as you need:

  1. wulfdist_ang15/js/wulf.angular.tlp.min.js: it includes all angular js files and all template files.
  2. wulfdist_ang15/js/wulf.angular.min.js: It includes all angular js files but no template files. it needs to work with template folder together.
  3. wulfdist_ang15/js/components/*/*.tlp.min.js: JS file + template file only for seperated component
  4. wulfdist_ang15/js/components/*/*.js: Only JS file for seperated component, it needs to work with template folder together.

Tips for using template folder if you choose option #2 or #4:
The templateUrl is relative path, that is wf/ng-template/*/*.html. Please copy the template folder to the correrct path if template is needed.
