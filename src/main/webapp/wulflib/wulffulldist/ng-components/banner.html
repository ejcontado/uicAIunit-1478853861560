Banner

Angular version of Banner component wfBanner, it contains four components wfBannerUtilitySet, wfBannerNavTabs, wfBannerActionSet and wfBannerNavTabContent. which are implemented by using .component() method introduced since Angular1.5. wfBannerNavTabs implemented nav Tab and Tab with menu(menuBar) function.

wf-banner settings

Attributes:Bindings Accept value Note
product-info: <
    product and Suite name info.
  • product-name - string
  • suite-name - string
  • search - boolea, if has search.
 The Product and suite name info, also include if search exist or not, like: productInfo = { product-name: "WebUI Look & Feel", suite-name: "(WULF)", search: true };
utility-set: <
    utility controls info array include following items:.
  • type - string, links or menu
  • name - string, name
  • link - string, url.
  • menuItems - string array for menu type, null for links type.
 The Utility controls info data like: utility-set= [ { type: "links", name: "User Name", link: "#", menuItems: [], disable: false }]
nav-tabs-info: < navigation control info:
    navTabIndex- active tab index; navigation Tab item array info which include:
  • label - string, links or menu
  • css-class - string, name
  • link - string, url.
  • data-item - string, associated nav tab item and tab content
  • menu-bar - string array for menu/sub menu items, only display for cssClass type of dropdown.
 nav-tabs-info ={ navTabIndex: -1, navTabs: [ { label: "item 1", cssClass: "n-banner-3Link", link: "#", dataItem: "item1", menuBar: "" }]
action-set: <
    action controls info array include following items:.
  • name - string, links or menu
  • css-class - "dropdown" or empty string, just one Action ,you can use empty string
  • icon - string, name
  • link - string, url.
  • has-child - string array for menu type, null for links type.
  • childItems - string, url.
 action-set = [ { name: "Manage", cssClass: "dropdown", icon: "icon icon-manage", link: "#", hasChild: true, childItems: [ {name: "sub item1", link: "#",selected: false}, {name: "sub item2", link: "#",selected: false} ] },
overflow-open: <
  • false
  • true
 overflow-open = true; if "overflowOpen = true",overflow function will open ,vice versa.

Keyboard accessibility

Besides the basic keyboard support, e.g. Tab, Shift + Tab and Enter. This component also supports the following keystrokes for the interaction.
Interaction Keystrokes Notes
Move focus to next/previous tab →/↓ or ←/↑ - next/previous tab
  • Using Tab or Shift + Tab can navigate out of the group of tabs
Select tab Spacebar/Enter - Select current focused tab
  • The associated tab content would be also shown
Move focus to next/previous menuBar ↓ or ↑ - next/previous menu item
  • Using Tab or Shift + Tab can navigate out of the group of tabs
Expand/Close sub menu → or ← - Expand/Close sub menu
  • Using Tab or Shift + Tab can navigate out of the group of tabs
Enter/Select dropdown menu Spacebar/Enter - Select current dropdown menu
  • The associated tab content would be also shown
Exit dropdown menu ESC - Exit current dropdown menu
  • Only on first level of sub menu items can exit.
