Supported Package Managers


You can use a package manager to ensure the right WULF version is in use. You find below the addresses for two such systems:

Bower
http://wulf-bower.china.nsn-net.net/

npm (DRAFT - setup ongoing)
http://wulf-npm.dynamic.nsn-net.net:4873

Benefit of this is that you don't need to download and unpack manually the WULF distribution into your development system. You can define easily which version of the WULF distribution you want to use in your project using bower.json file for bower and package.json file for npm. After that your compilation system can make sure that the right version is included to your web project.

Usage


Bower Configuration:

1.Point the bower register to our wulf-bower server
2.Add currect proxy setting to the rc file if you can't access the internet

Notice:
The config is obtained by merging multiple configurations by this order of importance:
CLI arguments via --config
Environment variables
Local .bowerrc located in the current working directory
All .bowerrc files upwards the directory tree
.bowerrc file located in user's home folder (~)
.bowerrc file located in the global folder (/)

Content of the .bowerrc (Generic Example)
{
    "registry": "http://wulf-bower.china.nsn-net.net/" ,
    "timeout": 300000,
    "proxy": "http://10.144.1.10:8080/",
    "https-proxy": "https://10.144.1.10:8080/",
    "no-proxy": "wulf-bower.china.nsn-net.net,gerrite1.ext.net.nokia.com"
}

You can add the WULF dependency inside the package manager's configuration file:

Bower:
save dependencies to your bower.json by bower install
     bower install wulf#1.2.0 --save
or
edit bower.json to add wulf to the dependency
"dependencies": {
    "wulf": "~1.2.0"
}

Check out the Bower web pages and npm web pages for more details how to utilize these systems.
