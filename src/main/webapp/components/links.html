JS & CSS dependency:

For links components, it will be ok to import following css file:
<link href="../wulfdist/css/wulf.basic.min.css" rel="stylesheet"/>
<link href="../wulfdist/css/wulf.links.min.css" rel="stylesheet">


    <!-- Normal text links -->
    <a href="http://google.fi" class="n-link">Normal link</a>
    <a href="http://google.fi" class="n-link n-link-visited">Visited link</a>
    <a href="http://google.fi" class="n-link n-link-disabled" disabled>Disabled link</a>
    


    <!-- Secondary text links -->
    <a href="http://google.fi" class="n-link n-link-secondary">Secondary link</a>
    <a href="http://google.fi" class="n-link n-link-secondary n-link-visited">Visited Secondary link</a>
    <a href="http://google.fi" class="n-link n-link-secondary n-link-disabled" disabled>Disabled secondary link</a>
    



    <!-- Call to action text links -->
    <a href="http://google.fi" class="n-link n-link-action">Call to action link</a><span class="n-link-action-icon"></span>
    <a href="http://google.fi" class="n-link n-link-action n-link-visited">Visited Call to action link</a><span class="n-link-action-icon"></span>
    <a href="http://google.fi" class="n-link n-link-action n-link-disabled" disabled>Disabled call to action link</a><span class="n-link-action-icon"></span>

Note1: (Chrome-only) if it is required to get rid of extra underline highlight in case of double-click done after the link/text on the same row, you need to wrap the link/text inside div.

Note2: Call-to-action disclosure icon is not acting as hyper link like it should.

Note3: It would be better to always add "disabled" property to disabled links especially in IE, however "n-link-disabled" class can work fine without "disabled" property in Chrome and FF.
